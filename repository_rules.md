# Repository Rules

## Default Workflow

- Start by checking `git status --short`; this repository commonly contains valuable untracked runtime state.
- Prefer `uv --cache-dir .uv-cache run ...` for Python commands so dependency behavior stays local and repeatable.
- Use `pytest` as the main verification command: `uv --cache-dir .uv-cache run pytest`.
- Keep edits scoped. This is a compact app, so small direct changes are usually better than new abstraction layers.
- Do not add a frontend build system unless the user explicitly asks for one.

## Files And State

- Treat source files as the tracked app: `main.py`, `auth.py`, `database.py`, `models.py`, `schemas.py`, `scripts/`, `templates/`, `static/css`, `static/js`, `data/drinks.json`, tests, and project metadata.
- Treat these as runtime/local state: `boozerun.db`, `test.db`, `users.json`, `static/uploads/`, `.venv/`, `.uv-cache/`, `.pytest_cache/`, `.playwright-cli/`, `.idea/`, and `__pycache__/`.
- Do not delete uploaded photos or local SQLite databases without direct user instruction.
- `data/wrapped.json` is generated by `scripts/build_wrapped_data.py`, but it is currently tracked. If changing Wrapped logic, regenerate it when the output is part of the requested change.
- `.gitignore` currently has a corrupted-looking `users.json` line with embedded NUL characters. Be careful when editing it, and verify `git status --short` afterward.

## Backend Rules

- `database.py` points the app at root `boozerun.db`; tests override this with root `test.db`.
- Schema changes need to account for three places: `models.py`, API serialization in `main.py`, and the simple migration path in `scripts/setup_db.py`.
- Keep the API response shapes stable unless tests and frontend consumers are updated together.
- For auth changes, update both `auth.py` and the `/token` flow in `main.py`; the frontend stores the access token in `localStorage` under `access_token`.
- Uploaded image handling belongs in `main.py` unless it grows enough to justify extracting a helper.

## Frontend Rules

- This is vanilla browser JavaScript. Use ES modules and existing module boundaries:
  - `static/js/modules/api.js` for network calls.
  - `static/js/modules/auth.js` for token and auth UI state.
  - `static/js/modules/map.js` for Leaflet and marker behavior.
  - `static/js/modules/ui.js` for rendering and UI helpers.
  - `static/js/app.js` for orchestration.
  - `static/js/wrapped.js` for the Wrapped reel only.
- When editing static JS/CSS, update the relevant cache-busting query strings in templates/imports:
  - `templates/index.html` for app CSS/JS.
  - `templates/wrapped.html` for Wrapped CSS/JS.
  - `static/js/app.js` for frontend module imports.
- CDN-backed dependencies mean browser behavior can depend on network access. Tests do not validate CDN loading.
- Keep the mobile-first behavior intact: geolocation must be user-triggerable, map detail sheets should remain touch-friendly, and leaderboard cards open user history.
- Use `@browser-use` / the Codex in-app browser for local app testing, visual inspection, screenshots, and interaction checks. Do not default to Playwright CLI for this repository.
- For local frontend checks, start Uvicorn, open `http://127.0.0.1:8000/` or `/wrapped` with `@browser-use`, reload after code changes, then verify the relevant DOM state or screenshot.

## Testing Rules

- Run the full suite after backend, script, data, or Wrapped changes:

```powershell
uv --cache-dir .uv-cache run pytest
```

- Expected current result: 10 tests pass with one Argon2 deprecation warning.
- Tests create and drop tables in `test.db`. Do not point tests at `boozerun.db`.
- For frontend or Wrapped changes, pair pytest with `@browser-use` inspection of the running app. Check both desktop and mobile-sized views when layout changed.
- Add or update tests when changing:
  - auth behavior,
  - entry creation or serialization,
  - leaderboard calculations,
  - database schema,
  - Wrapped generation.

## Running The App

- Local browser:

```powershell
uv --cache-dir .uv-cache run uvicorn main:app --host 127.0.0.1 --port 8000
```

- LAN/mobile testing:

```powershell
uv --cache-dir .uv-cache run uvicorn main:app --host 0.0.0.0 --port 8000
```

- HTTPS deployment with the local Caddy setup:

```powershell
.\caddy_windows_amd64.exe run
```

- For mobile GPS, use HTTPS or localhost. Plain HTTP on another device will usually block geolocation.

## Data And User Management

- To sync users from `users.json` and run the current manual migrations:

```powershell
uv --cache-dir .uv-cache run python scripts/setup_db.py
```

- `users.json` should contain:

```json
[
  {"username": "matei", "password": "securepassword"}
]
```

- To manage users without editing `users.json`, use `scripts/manage_users.py`.
- Deleting a user with entries requires `--delete-entries`; the script intentionally refuses otherwise.

## Wrapped Workflow

- Regenerate Wrapped:

```powershell
uv --cache-dir .uv-cache run python scripts/build_wrapped_data.py
```

- The generator validates image files relative to the project root and skips empty/missing images.
- If a Wrapped slide references specific entry IDs, verify those IDs exist in the active `boozerun.db`.
- After changing Wrapped templates, CSS, JS, or data generation, run tests and inspect `/wrapped` with `@browser-use` whenever possible.
